# スタイル設計(指定フォーマットへのその場対応)

ステータス: proposed decision / 最終更新: 2026-07-14

社用スライドなどで強制される特定のスタイル(色・フォント・ロゴ・フッター・表紙様式)を、**事前のテーマ整備なしに、渡されたフォーマットへその場で柔軟に合わせられる**ようにする設計。

前提となる判断(2026-07-14 確認):

- 実物の社用テンプレ(PowerPoint)があり、早めに使いたい
- 日本語フォントは Noto Sans CJK を標準とする
- **事前にテーマパックを整備・配布する方式は採らない。** スタイルはデッキごとに、その場で合わせる(本書の初版が提案したテーマパック案は「幾何まで変える本格テーマが必要になったら」の将来候補へ降格)

## 1. 原則: スタイルにも「サブセット + 生」の二層を適用する

本文の語彙と同じ構造をスタイルに持ち込む。

| 層 | 置き場 | ツールの理解 | プレビュー | PDF |
|---|---|---|---|---|
| **スタイル語彙**(§2) | `%% style` 領域 | 完全理解 | **即座に追従**(色・フォント・ロゴ・フッター) | 反映 |
| **生スタイル** | `%% preamble-extra` 領域 | 素通し | 反映されない(近似のまま) | **完全に反映** |

- スタイルは**デッキのソースに書く**。外部ファイル・レジストリ・インストールは不要。AI がその場で書け、diff で見え、git で一緒に運ばれる
- 本文の編集語彙(itemize・block・deckcanvas 等)はスタイルによって一切変わらない。AI・パーサ・SKILL.md の本文知識はスタイル非依存のまま
- 生スタイルの検証は本文の生ブロックと同じく TeX 側(`deck snapshot` / PDF)。「プレビューは意味の確認、PDF は体裁の確認」という役割分担(design.md 原則 5)をスタイルにも適用する

## 2. スタイル語彙 v1(`%% style` 領域)

ユーザー編集可能領域を 1 つ増やす。中身はツール管理プリアンブルが定義する `deck*` マクロ(予約名前空間・L016)なので、そのまま正しい LaTeX としてコンパイルできる。

```latex
%% style:begin
\deckcolor{structure}{0F62FE}
\deckcolor{alert}{DA1E28}
\deckfont{main}{Noto Sans CJK JP}
\deckfont{mono}{Source Han Code JP}
\decklogo[x=0.930,y=0.020,w=0.050]{assets/logo.pdf}
\deckfooter{ACME Corp. — Confidential}
%% style:end
```

| コマンド | 意味 | TeX 側の実装 | プレビュー側 |
|---|---|---|---|
| `\deckcolor{役割}{RRGGBB}` | 役割色の変更。役割: `structure` `alert` `example` `text` `background` | `\setbeamercolor` 群 | CSS 変数(`--deck-structure` 等) |
| `\deckfont{main\|mono}{名前}` | フォント指定(名前参照。§4) | fontspec / xeCJK | `font-family`(ローカルフォントへフォールバック) |
| `\decklogo[x=,y=,w=]{path}` | 全フレーム共通のロゴ。座標はキャンバスと同じ正規化値 | 背景テンプレートへの絶対配置 | 各スライドへ絶対配置 `<img>` |
| `\deckfooter{テキスト}` | フッター(左: テキスト、右: ページ番号) | 絶対配置のフッターバー | 各スライドへ絶対配置 |

設計上の制約: **v1 のスタイル語彙は幾何中立**(本文領域・タイトル帯の寸法を変えない)。ロゴとフッターは本文領域を消費しないオーバーレイとして実装する。これによりキャンバス座標系(subset-spec §2.8 の実測定数)とプレビューの幾何がスタイルに関わらず安定する。タイトル帯の高さやレイアウト自体を変える様式は v1 では生スタイル(preamble-extra)へ——プレビューの幾何がずれ得ることは既知の劣化として扱い、snapshot で検証する。

リント:

- **L020**: `%% style` 領域に語彙外の記述(不明なコマンド・不正な役割名・不正な色値・許容外のキー)。**error**(未定義マクロはコンパイルエラーになるため事前に検出する)
- ロゴ画像の存在は L004、`deck*` の再定義禁止は L016 をそのまま適用

## 3. 「この見本に合わせて」— AI によるその場対応

スタイル合わせを微調整モード(ai-protocol §7)と同格の通常タスクにする。事前準備は不要。

```
人間: この会社テンプレ(template.pptx / 見本.pdf / スクショ)に合わせて
エージェント:
  1. 見本から色(構造色・強調色)・フォント・ロゴ・フッター文言を抽出
     (pptx は zip+XML なので theme1.xml から色とフォントを機械抽出できる。
      画像見本なら vision で読み取る)
  2. ロゴ素材を assets/ へ配置し、%% style 領域を書く
  3. スタイル語彙で表現できない様式(表紙の帯・背景の飾り等)だけを
     preamble-extra に \setbeamertemplate 等で書き、使ったことを報告する
  4. deck snapshot の出力と見本を並べて比較し、2 回まで自律修正
  5. 報告: 反映した項目 / 生スタイルに落ちた項目 / lint・check 結果
```

- SKILL.md の指示パターン集に本パターンを追加する(ai-protocol §6)
- 同じ形式を再利用したいときは、`%% style` 領域 + preamble-extra + assets をコピーするだけでよい(スタイルの実体がソースにあるため)。頻用するならその雛形を `deck init` の選択肢に加える——これが「テーマ」の実体であり、専用の配布機構は要らない

## 4. フォントと日本語(CJK)

- 標準は **Noto Sans CJK JP**。`\deckfont` は名前参照であり、フォントバイナリはリポジトリに同梱しない
- 解決順: ローカルにあればそれを使い、無ければ `deck fonts fetch` が取得してキャッシュする(tectonic のパッケージ自動取得と同じ思想)。会社指定フォント(再配布不可が多い)は「名前参照 + 各自インストール」で扱い、不在は lint で検出
- プレビューは `font-family` のフォールバック指定で近似(プレビューは近似でよい)
- design.md §9 の CJK 未決事項はこの機構で解消する。default スタイルに Noto CJK を入れた時点で日本語 fixture を追加する

## 5. 実装への波及

| 箇所 | 内容 |
|---|---|
| subset-spec | `%% style` 領域と語彙表・L020(反映済み) |
| core(パーサ) | style 領域 → `DeckStyle` ノード(色マップ・フォント・ロゴ・フッター)。語彙外は L020 対象として保持 |
| ツール管理プリアンブル | `\deckcolor` 等の TeX 実装(deck-canvas-preamble と同じ方式で注入) |
| renderer | `DeckStyle` → CSS 変数 + ロゴ/フッターのオーバーレイ描画。幾何は `Theme`(theme.ts、実測定数)のままで独立 |
| ai-protocol / SKILL.md | §6 指示パターンに「見本に合わせて」を追加 |
